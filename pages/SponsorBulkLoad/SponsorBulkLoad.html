Sponsor Bulk User Loader

Upload Bulk File

• Upload a .txt file containing ONE command per line.
• Fields in each command are separated by "|".
• Commands are processed in order, top to bottom.
• Results appear live in the Successes and Errors panels.

COMMAND TYPES
────────────────────────────────
D||First|Last|Email
    → Adds a Driver to your organization.
    → Creates the user if they do not exist.
    → If user exists, attaches them to your organization.
    → If already attached, marked as success.
                                
S||First|Last|Email
    → Same as D, but adds a Sponsor instead.

PROCESSING RULES
────────────────────────────────
• Organization is inferred, specifying it will result in an error.
• Email addresses uniquely identify users.
• Blank lines are ignored.
• Invalid lines appear in Errors but do NOT stop processing.
• Large files are automatically broken into batches.

EXAMPLE
────────────────────────────────
D||Sarah|Connor|sconnor@mail.com
S||Kyle|Reese|kreese@mail.com
D||Ellen|Ripley|eripley@mail.com

SUMMARY OF EXAMPLE
────────────────────────────────
1. Add Sarah Connor as a Driver.
2. Add Kyle Reese as a Sponsor.
3. Add Ellen Ripley as a Driver.
